Engineering highlights

UI-thread animation

The balance counter tweens by driving a whitelisted TextInput.text prop from a Reanimated worklet — no JS-thread re-render per frame, so scrolling stays at 120Hz.

GPU charts (Skia)

Sparklines are cubic Skia paths with a gradient area fill, memoised against their inputs so a 30s market refetch never recomputes list geometry.

Server vs client state

TanStack Query owns network data (cache, retry, refetch, cancellation); Zustand owns only user data (holdings/watchlist) persisted via MMKV.

Pure, tested logic

Money math lives in pure selectors (selectPortfolioValue) unit-tested with Jest; CI runs typecheck → lint → test on every PR.

Offline-first

A deterministic fixture backs the API layer, so every screen stays interactive with no network or API key — resilient on flaky mobile connections.

Type-safety, maxed

strict, noUncheckedIndexedAccess, exactOptionalPropertyTypes, typed Expo Router routes and a typed useTheme().

A representative slice

src/components/AnimatedNumber.tsx — tween a currency value on the UI thread
export function AnimatedNumber({ value, style }: { value: number; style?: TextStyle }) {
  const v = useSharedValue(value);
  useEffect(() => {
    v.value = withTiming(value, { duration: 650, easing: Easing.out(Easing.cubic) });
  }, [value, v]);

  // runs as a worklet — formats & writes to the native text prop every frame, no JS re-render
  const props = useAnimatedProps(() => ({ text: formatUsd(v.value), defaultValue: formatUsd(v.value) }));
  return <AnimatedTextInput editable={false} animatedProps={props} style={style} />;
}
